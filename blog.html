var variable

var is a global scope variable. It can be defined once and used anywhere in the program. It is used in both function scope and block scope. The special feature of var is that it is hoisting. As a result, even if it is defined between the function scope and the block scope, it is hoisted up, thereby increasing the possibility of external access and errors in the program. This is why we usually do not declare var BF variables in programs.

let variable

let is a global variable. It is very similar to tie var. It can be defined once and used anywhere in the program. It is used in both function scope and block scope. But it doesn't do hoisting like var . As a result, unauthorized access from the outside is protected. As a result, we usually declare variables with let .

const variable

A const variable is an immutable variable. A const variable cannot be reassigned. This is a block scope variable. We usually assign a value that does not need to be changed to a constant variable. Accessing a field or element is often assigned to a constant variable while working on the website.

null & undefine

When we want to assign the value of a variable later or leave the value of the variable empty, we use null or undefine. Here both null and undefine are false or negative values.
